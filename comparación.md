          Comparación entre Programación Tradicional y Programación Orientada a Objetos (POO)

Ambos métodos funcionan, pero cada uno sirve mejor para distintos tipos de programas.
La programación tradicional es más simple para tareas pequeñas, mientras que POO es más útil cuando el programa 
necesita crecer o tener más organización.


1. Programación Tradicional

En la versión tradicional usé funciones normales.
Primero una función para ingresar las temperaturas día por día, y luego otra para calcular el promedio. El 
programa es más directo solo debo llamar a las funciones y listo.

Es fácil de escribir y de entender, creo que es ideal para programas pequeños.
Por otro lado, si el programa crece, se vuelve desordenado.


2. Programación Orientada a Objetos (POO)

Organicé el código de otra forma, primero se creó una clase que guarda las temperaturas y tiene métodos para pedir
los datos y calcular el promedio, haciendo el código más ordenado.

El código queda más limpio y organizado, es más fácil agregar cosas despues y la información queda “protegida” 
dentro de la clase.
Creo que es un poco complejo porque cuesta entender cómo dividir en clases, atributos y métodos, también requiere más 
estructura incluso para algo sencillo como promediar las temperaturas de 7 días.
